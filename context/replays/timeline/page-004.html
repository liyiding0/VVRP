Refine VVRP SOCK API to Linux-style unified socket functions

              
Changed SOCK public API to one Linux/BSD-style function family: socket, bind, connect, send, sendto, close. IPv4 remains the only implemented domain internally; IPv6 is represented by AF_INET6 but returns unsupported until implemented. Removed public IPv4-specific socket entry points.

Add FWD forwarding dispatch layer with Ethernet output handler

              
Added src/FWD as the forwarding dispatch layer below SOCK/FIB. FWD consumes FIBEntry, resolves the output IFNET interface, dispatches by interface kind, and implements Ethernet output with ARP adjacency lookup and injectable raw frame ports. PPP and other interface types are explicitly unsupported until handlers are added. All new FWD module functions and variables use FWD_ naming.
